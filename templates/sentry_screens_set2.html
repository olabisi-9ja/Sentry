Admin Access — Gated, Not a Tab

Admin is a separate, authenticated surface. It never appears in the public app's navigation —
reaching it requires a staff login, independent of anything a student can tap into.

Access requests are reviewed manually — this is not a self-serve signup.

Why gated, not a tab

Putting an "Admin" icon in the public bottom nav — even locked — tells every student it exists and invites probing. Removing it entirely is the safer default.

Role is checked server-side on every request, not just hidden in the UI — so this isn't security by obscurity, it's a genuinely separate authenticated surface.
